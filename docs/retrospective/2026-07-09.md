# Retrospective — 2026-07-09

## Objective

Harden the Bedrock-to-provider boundary without redesigning the library: make unsupported Bedrock
controls explicit and configurable per provider, preserve mixed tool-result content, and make
provider streaming/error behavior deterministic.

## Decisions

- Added provider-level `bedrock_controls.mode` with strict `fail` and compatibility
  `passthrough` modes; `fail` is the default.
- Kept accepted controls in canonical request extensions and enforced the policy at the common
  client/transport boundary.
- Used a dedicated `BedmockCompatibilityWarning` when passthrough accepts but drops controls.
- Defined the streaming retry boundary as the first canonical event yielded to the caller.
- Required a terminal `finish_reason`; malformed or incomplete streams use
  `ModelStreamErrorException`.
- Preserved residual text/image blocks when tool results become separate provider messages.
- Classified malformed successful provider responses and unknown internal failures as 500-class
  errors while preserving explicit 401/403 access-denied mapping.

## Changed surfaces

- Provider profile schema and loader.
- InvokeModel control validation and canonical request extensions.
- Shared client-to-transport compatibility policy.
- OpenAI-compatible non-streaming, streaming, retry, error, and tool-message mapping.
- ConverseStream event encoding.
- Public exceptions, configuration/compatibility documentation, examples, and unit tests.
- Approved implementation plan under `docs/plans/`.

## Failures and recovery

- One multi-file `apply_patch` failed on a later hunk after earlier hunks had already applied.
  Subsequent work treated patch application as potentially partial and re-inspected every target.
- A test helper was initially inserted inside another helper, causing several unrelated failures.
  Inspecting the failing source region exposed the indentation/scope error; the helper was moved to
  module scope before continuing.
- Ruff's default cache path was not writable in the sandbox. Running Ruff with `--no-cache`
  produced a reliable check and exposed the one file that needed formatting.

## Verification

- Targeted provider, routing, and facade regression suite: 42 passed after the final provider-schema
  classification fix.
- Full pytest: 66 passed.
- Ruff lint passed; Ruff format confirmed 85 files already formatted.
- Mypy passed for 51 source files.
- The sdist and wheel built successfully; `twine check` passed for both artifacts.
- `git diff --check` passed, and scans found no leftover task placeholders or live-looking secrets.

## Next steps

- Keep `supported_bedrock_controls` transport-specific as real mappings are added.
- If a provider documents a nonstandard in-stream error envelope, add a focused parser fixture
  rather than weakening the generic SSE rules.
- Consider extracting the streaming state machine into smaller private helpers only if another
  transport needs the same behavior; the current change intentionally avoids a broad abstraction.
